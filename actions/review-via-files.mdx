The Files view allows reviewers and client reviewers to take review actions directly from the file list. Instead of opening the regular review queue first, you can filter the files you need, open the available file actions, and Accept, Reject, or Edit based on your role.

### 1. Access Files View

* Log in to the Labellerr platform using your credentials.
* Open the required project from the dashboard.
* Navigate to the **Files** section.
* Apply file status filters such as review-ready or client-review-ready files to find the files that need action.

Once the filtered files appear, use the available action options from the Files view. The actions shown depend on your project role and the current status of each file and then click on any file link to open.

### 2. Role-Based Review Options

**Super Admin**

Super Admin users can view files across the project and perform all available review actions, including **Accept**, **Reject**, and **Edit**, wherever their permissions allow.

**Client Reviewer**

Client Reviewers can review files that are in the client review section or client-review-ready status. From the Files view, they can **Accept** files that meet the required quality standard or **Reject** files that need correction.

**Reviewer**

Reviewers can review files assigned to the review section or review-ready status. From the Files view, they can **Accept**, **Reject**, or **Edit** files before taking the final review action.


### 3. Available Actions

**Accept**

Use **Accept** when the file and its annotations meet the required quality standard. Accepting a file moves it forward in the workflow.

**Reject**

Use **Reject** when the file needs correction. Add relevant remarks or correction feedback so the next user understands what must be fixed.

**Edit**

Use **Edit** to open the file and update annotations before completing the review action. This is useful when a reviewer or Super Admin needs to make changes directly instead of sending the file back.
