## Creating a New Project

<Card title="Project Overview" icon="circle-info">
A **project** in Labellerr organizes your datasets and their annotations. Use the following method to create a project:

### Limitations:

*   Maximum of **2,500 files** per folder.
*   Total folder size should not exceed **2.5 GB**.

### Acceptable value:

*   **option_type**
    *   `'input'`, `'radio'`, `'boolean'`, `'select'`, `'dropdown'`, `'stt'`, `'imc'`, `'BoundingBox'`, `'polygon'`, `'dot'`, `'audio'`
</Card>

<Card title="Data Types and Supported Formats" icon="database">
The Labellerr SDK supports various data types and file formats for your annotation projects:

### **Supported Data Types and Extensions**

| Data Type | Description | Supported Extensions |
|---|---|---|
| image | Image files for visual annotation | .jpg, .jpeg, .png, .bmp, .tiff |
| video | Video content for temporal annotation | .mp4 |
| audio | Audio files for sound annotation | .mp3, .wav |
| document | Document files for text analysis | .pdf |
| text | Plain text files for text annotation | .txt |

When using the SDK methods, specify the data type as one of: `'image'`, `'video'`, `'audio'`, `'document'`, or `'text'`.

When using the `initiate_create_project()` function, you **must provide either**:

*   `annotation_guide` (as a dictionary), **or**
*   `annotation_template_id` (as a string)

**Important:** At least one of these two fields is required. If both are missing, the project creation will fail.
</Card>

<Card title="Example: Creating a Project with an Existing Template" icon="code">
```python Project Creation with Template lines icon="template-icon"
from labellerr.client import LabellerrClient
from labellerr.core.projects import create_project
from labellerr.exceptions import LabellerrError

# Initialize the client with your API credentials
client = LabellerrClient(api_key, api_secret, client_id)
project_payload = {
    'data_type': 'image',
    'created_by': 'user@example.com',
    'project_name': 'Image Classification Project',
    'annotation_template_id': 'existing_annotation_template_id',
    'rotation_config': {
        'annotation_rotation_count': 1,
        'review_rotation_count': 1,
        'client_review_rotation_count': 1
    },
    'autolabel': False,
    'folder_to_upload': 'path/to/your/dataset'
}
try:
    project = create_project(client, project_payload)
    print(f"Project created successfully. Project ID: {project.project_id}")
except LabellerrError as e:
    print(f"Project creation failed: {str(e)}")
```
</Card>

<Card title="Example: Creating a Project with Inline Annotation Questions" icon="code">
You can also add **classification questions and** [**annotation questions**](/sdk/add-annotation-questions) while creating the project.

```python Project Creation with Inline Annotation Guide lines icon="question"
from labellerr.client import LabellerrClient
from labellerr.core.projects import create_project
from labellerr.core.exceptions import LabellerrError

# Initialize the client with your API credentials
client = LabellerrClient(api_key, api_secret, client_id)

project_payload = {
    'data_type': 'image',
    'created_by': 'user@example.com',
    'project_name': 'Image Classification Project',
    'annotation_guide': [
        {
            "question": "What is the main object in the image?",
            "option_type": "dropdown",
            "required": True,
            "options": [
                {"option_name": "Car"},
                {"option_name": "Building"},
                {"option_name": "Person"}
            ]
        }
    ],
    'rotation_config': {
        'annotation_rotation_count': 1,
        'review_rotation_count': 1,
        'client_review_rotation_count': 1
    },
    'autolabel': False,
    'folder_to_upload': 'path/to/your/dataset'
}

try:
    project = create_project(client=client, payload=project_payload)
    print(f"Project created successfully. Project ID: {project.project_id}")
except LabellerrError as e:
    print(f"Project creation failed: {str(e)}")
```
</Card>

<Card title="Example: Creating a Project with Existing Datasets" icon="code">
You can create a project using datasets that have already been created separately. This is useful for reusing the same data across multiple projects.

```python Project Creation with Existing Datasets lines icon="database"
from labellerr.client import LabellerrClient
from labellerr.core.projects import create_project
from labellerr.core.exceptions import LabellerrError

# Initialize the client with your API credentials
client = LabellerrClient(api_key, api_secret, client_id)

project_payload = {
    'data_type': 'image',
    'created_by': 'user@example.com',
    'project_name': 'Multi-Dataset Project',
    'annotation_template_id': "existing_annotation_template_id",
    'datasets': ["dataset_id_1", "dataset_id_2"],  # Attach existing datasets
    'rotation_config': {
        'annotation_rotation_count': 1,
        'review_rotation_count': 1,
        'client_review_rotation_count': 1
    },
    'autolabel': False
}

try:
    project = create_project(client=client, payload=project_payload)
    print(f"Project created successfully. Project ID: {project.project_id}")
    print(f"Attached datasets: {project.attached_datasets}")
except LabellerrError as e:
    print(f"Project creation failed: {str(e)}")
```

<Note>
When using the `datasets` parameter, you should **not** include `folder_to_upload`, `files_to_upload`, `dataset_name`, or `dataset_description` in the payload. The datasets must already exist and contain files.
</Note>
</Card>

## Glossary

<Card title="Project Payload Fields" icon="list">
| Variable/Key | Type | Required | Description | Example Value |
|---|---|---|---|---|
| data_type | String | Yes | Type of data in the project | image, video, audio, document, text |
| created_by | String | Yes | Email address of the project creator | user@example.com |
| project_name | String | Yes | Display name for the annotation project | Image Classification Project |
| annotation_template_id | String | Conditional* | ID of existing annotation template | template_456 |
| annotation_guide | List | Conditional* | List of annotation questions (simplified structure) | See examples above |
| autolabel | Boolean | Yes | Enable/disable automatic AI pre-labeling | false |
| rotation_config | Object | No | Configuration for annotation and review rotations | See below |
| datasets | List | Conditional** | List of existing dataset IDs to attach | ["dataset_id_1", "dataset_id_2"] |
| folder_to_upload | String | Conditional** | Local file path to dataset folder | path/to/your/dataset |
| files_to_upload | List | Conditional** | List of specific file paths to upload | ["file1.jpg", "file2.jpg"] |

**Conditional Requirements:**

\* Either `annotation_template_id` OR `annotation_guide` must be provided (not both)

\*\* Either `datasets` (for existing datasets) OR `folder_to_upload`/`files_to_upload` (for new dataset creation) must be provided

**Rotation Config Structure:**

| Field | Type | Required | Default | Description |
|---|---|---|---|---|
| annotation_rotation_count | Integer | No | 1 | Number of times each item needs to be annotated |
| review_rotation_count | Integer | No | 1 | Number of times annotated items need to be reviewed |
| client_review_rotation_count | Integer | No | 1 | Number of times client review needs to be performed |
</Card>
