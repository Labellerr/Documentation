<Card title="Introduction" icon="circle-info">
  This guide will help you understand how to write your own annotation questions in JSON format, whether they are for drawing things on images (like bounding boxes or dots) or for selecting/entering answers (like dropdowns or booleans).

  We will explain:

  * Types of annotation questions
  * Required and optional fields
  * Examples for both types

## Glossary (JSON Key Explanations)

| Title | Description |
|---|---|
| question_number | Sequential number for the question (e.g., 1, 2, 3). |
| question_id | Unique identifier (UUID) for the question. |
| question | The text of the question or label name. |
| option_type | Type of annotation: "polygon", "BoundingBox", "dropdown", "select", "radio", "boolean", etc. |
| class | Label or name of the object (e.g., "Car", "Tree"). |
| objectType | Shape used for annotation ("BoundingBox", "polygon", "dot", "polyline"). |
| dataset | Type of dataset: "image", "video", "audio", etc. |
| color | Display color in hex code (e.g., "#FF0000" for red). |
| uniqueKey | Unique identifier (UUID) for the object or question. |
| attributes | Optional nested questions or features related to the object. |
| allowAnnotationToBeMarkedIncorrect | Whether the object can be marked as incorrectly drawn (true/false). |
| customer_question_id | ID from an external system (leave null if not used). |
| mode | Usually "edit" – defines how the annotation appears in the UI. |
| key | Unique identifier for classification questions. |
| componentClass | The actual text or label of the question shown to users. |
| annotation | Input type: "dropdown", "select", "radio", or "boolean". |
| required | Whether answering this question is mandatory (true/false). |
| options | List of selectable answers (each with key, value, and optional color). |
| sort | Whether to display the options in alphabetical order. |
| searchBar | Adds a search box to the dropdown/select if there are many options. |
| defaultValue | The answer selected by default if the user doesn't pick one. |
| notSureRemark | Adds a "Not Sure" or similar fallback option. |
| questionGuidelines | Extra instructions shown with the question to help users. |
| question_metadata | System metadata, usually left null. |
| autolabel | Configuration for ML model-based label suggestions. |

## 1. Types of Annotation Questions

<Card title="Supported Annotation Types by Dataset" icon="database">
Different datasets support different types of annotations:

### Image

* Object-Based: BoundingBox, polygon, dot, polyline  
* Classification: All types (dropdown, boolean, radio, select)

### Document (Docs)

* Only supports Classification annotations

### Video

* Object-Based: generic, BoundingBox, dot, polygon  
* Classification: All types

### Audio

* Object-Based only using objectType: audio

```json Audio Object-Based Annotation Example lines icon="volume-high"
{
  "color": "#5F9EA0",
  "class": "ab",
  "uniqueKey": "<object_unique_key>",
  "objectType": "audio",
  "attributes": [],
  "allowAnnotationToBeMarkedIncorrect": false,
  "dataset": "audio"
}
```

* Classification: Same format as other datasets

### Audio Text / PDF

* Only supports Classification annotations
</Card>

<Card title="Main Annotation Question Types" icon="layout-template">
There are <strong>two main types of annotation question</strong> that you can create:

**A. Object-Based** (e.g., draw bounding boxes, polygons, dots, polylines)  
Used when you want users to draw or mark things directly on an image.

**B. Classification** (e.g., dropdowns, booleans, radio buttons)  
Used when users answer a question based on the image without drawing.

## 2. Object-Based Questions

<Card title="Required Fields for Object-Based Questions" icon="asterisk">
  
| Field | Description |
|---|---|
| class | The label or name of the object (e.g., “Car”) |
| objectType | Shape of the object: BoundingBox, polygon, dot, polyline |
| dataset | Always set to image / video / audio |

</Card>

<Card title="Optional Fields for Object-Based Questions" icon="option">
  
| Field | Description |
|---|---|
| color | Color code to display the object (e.g., "#FF0000") |
| uniqueKey | Unique ID for the object (auto-generated if not given) |
| attributes | Extra details for the object. See example below for full JSON structure. |
| allowAnnotationToBeMarkedIncorrect | Allow marking if drawn incorrectly (default: false) |
| customer_question_id | If you’re linking to another system, otherwise leave null |
| mode | Display mode, usually “edit” |

</Card>

```json Attributes Field Example lines icon="list-tree"
{
  "color": "#00CED1",
  "class": "obj3",
  "uniqueKey": "<object_unique_key>",
  "objectType": "polyline",
  "attributes": [
    {
      "key": "<unique_id>",
      "componentClass": "obj_3_class",
      "annotation": "select",
      "required": false,
      "options": [
        {
          "key": "<option_1_id>",
          "value": "ab",
          "color": "#343df3"
        },
        {
          "key": "<option_2_id>",
          "value": "cd",
          "color": "#a09060"
        }
      ],
      "type": "attribute",
      "sort": false,
      "searchBar": false,
      "defaultValue": false,
      "notSureRemark": null,
      "questionGuidelines": ""
    }
  ]
}
```

## 3. Classification Questions

<Card title="Classification Annotation Types" icon="list-check">
Used for selecting or entering answers. Valid annotation types:

* <code>boolean</code> (Yes/No , True/False , Yes/No/Not Sure)  
* <code>radio</code> (Multiple options, one selection)  
* <code>dropdown</code> (Multiple options in a dropdown)  
* <code>select</code> (Alternate dropdown)

<Note>
  <code>annotation</code> (for classification questions) must be one of: <code>dropdown</code>, <code>select</code>, <code>radio</code>, or <code>boolean</code>.
</Note>
</Card>

<Card title="Required Fields for Classification Questions" icon="asterisk">
  
| Field | Description |
|---|---|
| key | A unique identifier (string) |
| componentClass | A short description or title of the question |
| annotation | Input type: must be dropdown, select, radio, or boolean |

</Card>

<Card title="Optional Fields for Classification Questions" icon="option">
  
| Field | Description |
|---|---|
| required | Whether the question must be answered (true or false) |
| options | List of selectable answers |
| sort | If true, options will appear sorted alphabetically |
| searchBar | Adds a search bar to options if there are many choices |
| defaultValue | The default selected answer (if any) |
| notSureRemark | Add an option like “Not sure” |
| questionGuidelines | Helpful instructions or tips shown to the user |
| question_metadata | Extra system data, usually left null |
| autolabel | Auto-suggestion model config (used for ML-based predictions) |

</Card>

<Card title="Boolean Example" icon="toggle-on">

## 4. How to Structure the JSON File

<Card title="Required Fields for SDK Questions" icon="asterisk">
When using the SDK to create annotation guidelines, each question must include:

| Field | Description | Required |
|---|---|---|
| question_number | Sequential number for the question (1, 2, 3, etc.) | Yes |
| question_id | Unique identifier (UUID) for the question | Yes |
| question | The text of the question or label name | Yes |
| option_type | Type of annotation (polygon, BoundingBox, dropdown, select, radio, boolean, etc.) | Yes |
| required | Whether the question must be answered (true/false) | Yes |
| options | List of options for the question | Yes |

<Note>
  Make sure to import `uuid` and use `str(uuid.uuid4())` to generate unique question IDs.
</Note>
</Card>

<Card title="Example File with Multiple Questions" icon="file-code">
Put all your questions in a list `[]`. You can mix object and classification questions.

```python Multiple Questions Example lines icon="list-check"
from labellerr.client import LabellerrClient
from labellerr.core.projects import create_annotation_guideline
import uuid

questions = [
  {
    "question_number": 1,
    "question": "Test",
    "option_type": "polygon",
    "required": True,
    "question_id": str(uuid.uuid4()),
    "options": [
      { "option_name": "#fe1236" }
    ]
  },
  {
    "question_number": 2,
    "question": "Test2",
    "option_type": "BoundingBox",
    "required": True,
    "question_id": str(uuid.uuid4()),
    "options": [
      { "option_name": "#afe126" }
    ]
  },
  {
    "question_number": 3,
    "question": "Test-Input",
    "option_type": "input",
    "required": True,
    "question_id": str(uuid.uuid4()),
    "options": []
  },
  {
    "question_number": 4,
    "question": "Multi-Test",
    "option_type": "select",
    "required": True,
    "question_id": str(uuid.uuid4()),
    "options": [
      { "option_name": "A" },
      { "option_name": "B" },
      { "option_name": "C" }
    ]
  },
  {
    "question_number": 5,
    "question": "Test-Dropdown",
    "option_type": "dropdown",
    "required": True,
    "question_id": str(uuid.uuid4()),
    "options": [
      { "option_name": "Sample A" },
      { "option_name": "Sample B" }
    ]
  },
  {
    "question_number": 6,
    "question": "Radio test",
    "option_type": "radio",
    "required": True,
    "question_id": str(uuid.uuid4()),
    "options": [
      { "option_name": "1" },
      { "option_name": "2" }
    ]
  }
]

# Create annotation guideline as a standalone template
client = LabellerrClient(api_key, api_secret, client_id)
template_id = create_annotation_guideline(
    client=client, 
    questions=questions, 
    template_name='Testing-123', 
    data_type='image'
)
print(f"Annotation template created with ID: {template_id}")
```

### Single Question Example

Here's a simple example with a single polygon question for video data:

```python Single Question Example lines icon="play"
from labellerr.core.projects import create_annotation_guideline
import uuid

questions = [
  {
    "question_number": 1,
    "question": "Test_12345",
    "option_type": "polygon",
    "required": True,
    "question_id": str(uuid.uuid4()),
    "options": [
      { "option_name": "#FE1236" }
    ]
  }
]

template_id = create_annotation_guideline(
    client=client,
    questions=questions,
    template_name='SDK_template',
    data_type='video'
)

print(f"Annotation template created with ID: {template_id}")
```
</Card>

<Note>
  `question_metadata` denotes the attributes for object type questions (Bounding Box and Polygons). The structure of `question_metadata` is:

  ```json question_metadata Structure lines icon="sitemap"
  {
    "question_metadata": [
      {
        "key": "<unique id>",
        "componentClass": "<name of the attribute>",
        "annotation": "<type of the attribute>",
        "required": "<boolean value>",
        "searchBar": "<boolean value>",
        "type": "attribute",
        "options": [
          { "key": "<unique id>", "value": "<option name>" }
        ]
      }
    ]
  }
  ```

  For example:

  ```json question_metadata Example lines icon="sitemap"
  {
    "question_metadata": [
      {
        "key": "<unique_id>",
        "componentClass": "Select the region of interest: ROI",
        "annotation": "select",
        "required": true,
        "options": [
          {
            "key": "<option_1_id>",
            "value": "maize - Spotted stem borer",
            "color": "#be410c"
          },
          {
            "key": "<option_2_id>",
            "value": "maize - Spotted stem borer",
            "color": "#9203a0"
          }
        ],
        "type": "attribute",
        "searchBar": true
      }
    ]
  }
  ```
</Note>
